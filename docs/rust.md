# Rust guide

The core crate is **`pedersen-kit`** — a pure-Rust library with the generic engine, the four axes, and ready-made instances.

!!! tip "API reference"
    The full rustdoc API lives at [**/api/**](../api/pedersen_kit/index.html) (generated by `cargo doc` and deployed alongside this site).

## Installing

Once published:

```toml
[dependencies]
pedersen-kit = "0.1"
```

Until [`ark-babyjubjub`](https://github.com/arkworks-rs/algebra/pull/1123) is on crates.io, pin the arkworks graph to its PR commit (see [Publishing](publishing.md)). Your workspace root needs the matching `[patch.crates-io]` block.

## The byte-compatible hashers

```rust
use pedersen_kit::circom::BabyJubjubCircom;

// Reusable: generators are derived once and cached across calls.
let mut h = BabyJubjubCircom::new();
let digest: [u8; 32] = h.hash(b"Hello");   // == circomlibjs pedersen.hash("Hello")
```

```rust
use pedersen_kit::zcash::JubjubSapling;

let mut z = JubjubSapling::new();                        // default "Zcash_PH"
// or a custom 8-byte personalization:
let mut z2 = JubjubSapling::with_personalization(*b"MyDomain");
let u = z.hash(b"Hello");                                // the u-coordinate (a field element)
```

## The arkworks-shaped instances

```rust
use pedersen_kit::instances::{BabyJubjubPedersen, JubjubBoweHopwood};

let p = BabyJubjubPedersen::build(64, 4);        // 64 windows × 4 bits = 256-bit capacity
let point = p.hash(b"data");                     // affine point

let b = JubjubBoweHopwood::build(16, 40);        // 16 × 40 3-bit chunks
let x = b.hash(b"data");                         // x-coordinate
```

These use reproducible `Deterministic` generators (not any spec's), so they're sound hashes with the right *structure* but are not byte-identical to a particular ecosystem.

## Building your own from the skeleton

```rust
use pedersen_kit::{Pedersen, Parameters, Unsigned, LsbFirst, WholePoint};

// Feed any Vec<Vec<C>> of precomputed generator powers.
let params = Parameters::adopt(my_generators);
let hasher = Pedersen::<C, Unsigned, LsbFirst, WholePoint>::from_params(params);
```

Mix and match `Encoding` / `BitLayout` / `Generators` / `OutputEncoding` to target other conventions.

## Features

Both are **on by default**:

| Feature | Enables | Pulls |
|---|---|---|
| `circom` | `pedersen_kit::circom` (byte-compatible circom instance) | `blake-hash` |
| `zcash`  | `pedersen_kit::zcash` (byte-compatible Zcash instance) | `blake2s_simd` |

`--no-default-features` gives the lean core (`Unsigned` / `BoweHopwood` over the arkworks curves) with **no** blake dependencies.

## Testing

```bash
cargo test -p pedersen-kit                        # parity + circom/zcash vectors + drift tests
cargo test -p pedersen-kit --no-default-features  # lean core only
```
